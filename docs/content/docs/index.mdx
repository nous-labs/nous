# @nouslabs/sdk

> forget what you know

Welcome to @nouslabs/sdk documentation. This guide will teach you everything you need to build applications on the Qubic blockchain, starting from the absolute basics.

## What is This?

@nouslabs/sdk is a TypeScript SDK for building applications on Qubic. Whether you're building a wallet, a trading dashboard, or integrating Qubic into your existing application, this SDK provides everything you need.

**What you can do with this SDK:**

- Query blockchain data (balances, transactions, network status)
- Authenticate users (multiple wallet types supported)
- Send transactions and interact with smart contracts
- Build real-time applications that respond to blockchain events
- Create and manage Qubic identities

**Where it works:**

- Modern browsers (Chrome, Firefox, Safari, Edge)
- Node.js applications (version 18+)
- Bun and Deno runtimes
- React, Vue, Angular, and other frameworks

## Quick Start (5 Minutes)

If you want to see it working immediately, here's a complete example:

### 1. Install

```bash
npm install @nouslabs/sdk
```

### 2. Query the Blockchain

```typescript
import { createQubicClient } from '@nouslabs/sdk';

// Create client
const qubic = createQubicClient();

// Get current network tick
const { tickInfo } = await qubic.live.getTickInfo();
console.log(`Current tick: ${tickInfo.tick}`);

// Get an account balance
const identity = 'BZVMIJXDWZQJWTFVEBPCJVFZDHXICRCLUVDUPKQGIJAFLEZCMMLUQHTXEXWA';
const { balance } = await qubic.live.getBalance(identity);
console.log(`Balance: ${balance.balance} QUBIC`);
```

### 3. What Just Happened?

- `createQubicClient()` - Creates a client connected to Qubic mainnet
- `qubic.live` - Accesses the real-time API client
- `getTickInfo()` - Queries the current blockchain state
- `getBalance()` - Checks how much QUBIC an address holds

## SDK Architecture

The SDK is organized into three main clients, each serving a specific purpose:

```typescript
const qubic = createQubicClient();

// Three specialized clients
qubic.live      // Real-time data and broadcasting
qubic.query     // Historical data and analytics
qubic.archive   // Legacy archive access
```

**QubicLiveClient** (`qubic.live`)
- Get current network status
- Query balances and assets
- Broadcast transactions
- Interact with smart contracts

**QueryClient** (`qubic.query`)
- Search transaction history
- Filter and paginate results
- Query tick archives
- Get computor information

**ArchiveClient** (`qubic.archive`)
- Access legacy archive data
- Historical blockchain queries
- Some methods deprecated (use QueryClient instead)

## Key Concepts

Before diving in, understand these core concepts:

### Identity

A Qubic identity is a 60-character uppercase string representing an account:

```typescript
const identity = 'BZVMIJXDWZQJWTFVEBPCJVFZDHXICRCLUVDUPKQGIJAFLEZCMMLUQHTXEXWA';
```

### Tick

Qubic uses "ticks" instead of blocks. A tick is created approximately every second:

```typescript
const { tickInfo } = await qubic.live.getTickInfo();
console.log(`Tick ${tickInfo.tick} in epoch ${tickInfo.epoch}`);
```

### Transaction

Transactions transfer QUBIC or invoke smart contract functions:

```typescript
const { transactions } = await qubic.query.getTransactionsForIdentity(identity);
transactions.forEach(tx => {
  console.log(`${tx.source} -> ${tx.destination}: ${tx.amount} QUBIC`);
});
```

### Smart Contract

Smart contracts are on-chain programs you can interact with:

```typescript
import { queryQX } from '@nouslabs/sdk';

// Query the QX exchange contract
const result = await queryQX(qubic.live, 1, { entityId: 123 });
```

## TypeScript Support

The SDK is written in TypeScript and provides full type safety:

```typescript
import type { Transaction, Balance, TickInfo } from '@nouslabs/sdk';

// All types are exported and documented
const processTransaction = (tx: Transaction) => {
  // TypeScript knows all available properties
  console.log(tx.source);       // string
  console.log(tx.destination);  // string
  console.log(tx.amount);       // number
  console.log(tx.status);       // TransactionStatus
};
```

Enable strict mode in your `tsconfig.json` for the best experience:

```json
{
  "compilerOptions": {
    "strict": true,
    "moduleResolution": "bundler",
    "target": "ES2022"
  }
}
```

## What's Different About Qubic?

If you're coming from other blockchains, here are key differences:

**Ticks vs Blocks**
- Qubic uses "ticks" (approximately 1 per second)
- No mining or proof-of-work
- Deterministic consensus

**Identity Format**
- 60-character uppercase strings
- No checksums (built into protocol)
- Derived from 55-character lowercase seeds

**Smart Contracts**
- Called "smart contracts" or "SCs"
- Different contract invocation pattern
- Query-based interaction model

**No Gas Fees**
- Transactions are free
- Limited by tick capacity
- Priority based on proof-of-work
