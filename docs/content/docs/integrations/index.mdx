# React Integrations

Complete React integration for Qubic blockchain with context providers, React Query hooks, and WalletConnect support.

## Integration Options

### 1. Basic React Provider

Use the simple `QubicProvider` for direct API access:

```tsx
import { QubicProvider, useQubic } from '@nvlabs/qts/react';

function App() {
  return (
    <QubicProvider>
      <YourApp />
    </QubicProvider>
  );
}

function Component() {
  const { live, query, archive } = useQubic();
  // Direct client access
}
```

**Best for:** Small apps, prototypes, manual state management

[→ Learn more about React Providers](./react-simple)

### 2. React Query Integration

Use React Query hooks for automatic caching, background refetching, and optimized data fetching:

```tsx
import { QubicQueryProvider, useCurrentTick, useBalance } from '@nvlabs/qts/react/query';

function App() {
  return (
    <QubicQueryProvider>
      <YourApp />
    </QubicQueryProvider>
  );
}

function Dashboard({ address }) {
  const { data: tick, isLoading } = useCurrentTick();
  const { data: balance } = useBalance(address);
  
  return <div>Tick: {tick?.tickInfo.tick}</div>;
}
```

**Best for:** Production apps, complex state, real-time data, large applications

[→ Learn more about React Query Hooks](./react-query)

### 3. Smart Contract Hooks

Ready-made hooks for all Qubic smart contracts (QX, Qswap, Qearn, etc.):

```tsx
import { useQXFees, useQswapPoolBasicState } from '@nvlabs/qts/react/query';

function TradingDashboard() {
  const { data: qxFees } = useQXFees();
  const { data: pool } = useQswapPoolBasicState(issuer, assetName);
  
  return <div>Trade Fee: {qxFees?.tradeFee}</div>;
}
```

**Best for:** DeFi apps, trading interfaces, contract interactions

[→ Learn more about Contract Hooks](./react-query-contracts)

## WalletConnect Integration

**Built-in Provider**: The SDK includes `WalletConnectProvider` and `useWalletConnect` hook - no manual setup required!

[→ Complete Next.js + WalletConnect + React Query Guide](./nextjs-walletconnect)

Simple integration for wallet connections:

```tsx
import { WalletConnectProvider, useWalletConnect } from '@nvlabs/qts/react';

function WalletButton() {
  const { adapter, accounts, isConnected } = useWalletConnect();
  
  const handleConnect = async () => {
    if (!adapter) return;
    const connection = await adapter.connect();
    await connection.approve();
  };
  
  return isConnected ? (
    <button onClick={() => adapter.disconnect()}>Disconnect</button>
  ) : (
    <button onClick={handleConnect}>Connect Wallet</button>
  );
}

function App() {
  return (
    <WalletConnectProvider 
      options={{
        projectId: 'YOUR_PROJECT_ID',
        metadata: { name: 'Your App', /* ... */ }
      }}
    >
      <YourApp />
    </WalletConnectProvider>
  );
}
```

</Callout>

## When to Use Each Approach

### Use Basic React Provider When:
- ✅ Building a prototype or small app
- ✅ You want minimal dependencies
- ✅ You're comfortable with manual state management
- ✅ You don't need automatic refetching
- ✅ Simple read operations only

### Use React Query When:
- ✅ Building a production application
- ✅ You need real-time data updates
- ✅ Complex data fetching patterns
- ✅ Multiple components accessing same data
- ✅ You want automatic caching and optimization
- ✅ Working with smart contracts extensively

## Installation

All React integrations are included in the main package:

```bash
npm install @nvlabs/qts

# For React Query integration, also install:
npm install @tanstack/react-query
```

## TypeScript Support

Both approaches have full TypeScript support with automatic type inference:

```tsx
import { useQubic } from '@nvlabs/qts/react';
import { useBalance } from '@nvlabs/qts/react/query';

// Types are automatically inferred
function Component() {
  const { live } = useQubic();
  // live is typed as QubicLiveClient
  
  const { data } = useBalance('address');
  // data is typed as GetBalanceResponse | undefined
}
```
